  1. What is a side-effect function?
  • A side-effect function is a function where some output or other task is presented but nothing is done to the data. Print(), cat(), and plot() are examples of side-effect functions.
  1. Why can you name a variable sd in a function and not cause any issues with the sd function?
  • A variable can be named sd within a function and not cause issues with the sd function because the sd within your function is local to your function’s environment while the sd function exists within the stats package which is a parent environment.

Create a wrapper function. Function has two if/else statements. The first if/else checks if the passed functions are atomic, numeric vectors. If either of the passed arguments are not one of those three conditions then it will stop running and return an error stating which condition it did not meet. The second if/else statement determines which function, getMAE() or getRMSE(), to call. If it is not specified then it will return both functions. 

my_wrapper<- function(vec1,vec2,fun = NULL,...){
  if(!is.vector(vec1) | !is.vector(vec2)){
    stop("Error: One or more arguments is not a vector")
  }
  else if(!is.numeric(vec1) | !is.numeric(vec2)){
    stop("Error: One or more arguments is not numeric")
  }
  else if(!is.atomic(vec1) | !is.atomic(vec2)){
    stop("Error: One or more arguments is not atomic")
  }
  
  if(!is.null(fun)){
    my_fun <- get(fun)
    cat(gsub("get",'',fun), my_fun(vec1,vec2,...),"\n\n")
    }
  else{
    return_val<-list(getMAE(vec1,vec2,...),
                     getRMSE(vec1,vec2,...))
    names(return_val)<-c("MAE","RMSE")
    return(return_val[])
  }
}

This chunk creates a function to search the newapi. The required argument is a keyword to search with date and APIkey set as defaults. The function creates a url then parses the returned JSON file and stores the results as a tibble which is then displayed for the user. 

search_NewsAPI <- function(title, date=today("UTC"), APIkey = source("api_key.r")){
  # Store title as a character string named keyword
  keyword<-as.character(title)
  # Store date is character string
  fromTime<-as.character(date)
  # Store today's date as character
  toTime<-as.character(today())
  # Pull APIKey for default
  APIkey<-as.character(APIkey$value)
  # Create URL
  .x<- httr::GET(paste0("https://newsapi.org/v2/everything?q=",keyword,"&from=",fromTime,"to=",toTime,"&apiKey=",APIkey))
  # Parse data and store as a tibble
  .d<-as_tibble(fromJSON(rawToChar(.x$content)))
  # Drop unnecessary variable
  .d<-.d$articles|>
    select(-source)
  # Return Tibble
  return(knitr::kable(.d[1:100,1:4],"simple"))
}
